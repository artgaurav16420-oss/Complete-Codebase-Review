# Security Policy

## Supported Versions

We release patches for security vulnerabilities in the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

We take the security of this project and its users seriously. If you discover a security vulnerability, please report it responsibly.

**Do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please report by opening a GitHub Security Advisory at:
https://github.com/artgaurav16420-oss/Complete-Codebase-Review/security/advisories/new

### What to include

- Type of issue (e.g., Remote Code Execution, SQL Injection, Cross-Site Scripting)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

### What to expect

- **Acknowledgment**: You will receive a reply within 48 hours confirming receipt.
- **Status updates**: We will provide updates every 7 days until the issue is resolved.
- **Disclosure coordination**: We will work with you to determine an appropriate disclosure timeline.

## Disclosure Policy

We follow a coordinated disclosure process:

1. **Report received** — triaged and confirmed within 48 hours.
2. **Patch development** — a fix is prepared and tested.
3. **Embargo period** — typically 30 days from patch availability to give users time to update.
4. **Public disclosure** — a security advisory is published on GitHub once the fix is released.

We ask that reporters refrain from public disclosure until an advisory is published. We will credit reporters in the advisory unless they request anonymity.

## Scope

This security policy covers the code review tool itself — its source code, dependencies, and build pipeline. It does not cover security findings *reported by* the tool against third-party codebases.

## Safe Harbor

We consider research and disclosure conducted in accordance with this policy as:

- Authorized conduct under applicable anti-hacking laws
- Exempt from the project's usual code-of-conduct restrictions on security research

We will not pursue legal action against researchers acting in good faith who follow this policy.
